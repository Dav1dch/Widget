# 当日进度Widget - 快速开始指南

## 快速测试步骤

### 第一步：同步项目
在Android Studio中：
1. 打开项目
2. 等待Gradle同步完成
3. 如果没有自动同步，点击 `File -> Sync Project with Gradle Files`

### 第二步：构建项目
1. 点击菜单栏 `Build`
2. 选择 `Make Project`
3. 等待构建完成（可能有R类的警告，这是正常的）

### 第三步：运行应用
1. 连接Android设备或启动模拟器
2. 点击绿色三角形Run按钮
3. 或使用快捷键 `Shift + F10`
4. 等待应用安装并启动

### 第四步：添加Widget到桌面

#### 方法A：长按桌面（推荐）
1. 长按设备桌面空白区域
2. 在弹出的菜单中选择"小组件"或"Widgets"
3. 滚动找到"当日进度"
4. 长按"当日进度"Widget
5. 拖拽到桌面合适位置
6. 松开手指

#### 方法B：通过应用列表
1. 打开应用抽屉
2. 找到"小组件"应用
3. 点击打开
4. 搜索或浏览找到"当日进度"
5. 点击添加
6. 选择尺寸（当前只有2x2）
7. 确认添加

### 第五步：验证功能

#### 检查显示
- [ ] 显示当前日期（如"周三 01/15"）
- [ ] 显示当前时间（如"14:30"）
- [ ] 显示进度条
- [ ] 显示进度百分比（如"60%"）
- [ ] 显示状态文字（如"工作中 已用 9.5小时"）

#### 测试更新
- [ ] 等待1分钟，观察时间是否更新
- [ ] 观察进度条是否变化
- [ ] 点击Widget，确认可以打开主应用

#### 测试工作日
- [ ] 周一到周五显示工作状态
- [ ] 8:00-18:00显示"工作中"
- [ ] 0:00-8:00显示"未上班"
- [ ] 18:00-24:00显示"已下班"

#### 测试休息日
- [ ] 周六、周日显示"休息日"
- [ ] 进度按24小时计算

## 常见问题

### Q1: 找不到Widget？
**A**: 尝试以下方法：
1. 重启设备
2. 重新安装应用
3. 尝试不同的桌面启动器
4. 清除应用数据后重新安装

### Q2: Widget显示异常？
**A**:
1. 检查应用是否正常运行
2. 查看Logcat日志
3. 清理项目后重新构建（`Build -> Clean Project`）

### Q3: 进度条不显示？
**A**: 这是RemoteViews的限制，进度条可能显示为系统默认样式，功能正常。

### Q4: 时间不准确？
**A**: 检查设备系统时间设置。
